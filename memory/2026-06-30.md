### 问题
- morning-report-briefing-round1/round2、world-cup-2026-daily-report 三个 cron 连续失败
- 错误：`LLM request failed.` / `Request was aborted`

### 根因
- **thinking: "medium"** 时 reasoning token 阶段长时间无可见产出 → OpenClaw stuck session recovery 在 ~360s 判定 stalled → `abort_embedded_run`

- Fallback 未生效原因：watchdog abort 后 failover 判定不可重试

### 修复
1. 三个 cron thinking 改为 `"off"`
2. `stuckSessionAbortMs` 从 360000 → 600000（10分钟），写入 `openclaw.json`
3. 网关已 `sudo systemctl restart` 生效
4. Fallback 模型：glm-5.2（已配置，watchdog 场景下仍不生效）
